# 06. fMRI Denoiser — Shiny App for Confound Selection + fsl_regfilt

_LC 2026-04-16_

---

## Concept

After fMRIprep, each subject has:
- A preprocessed BOLD in MNI space: `*_desc-preproc_bold.nii.gz`
- A confounds TSV from fMRIprep: `*_desc-confounds_timeseries.tsv` (motion params, aCompCor, scrubbing flags, cosine HPF at 128s, ...)
- A custom cosine HPF TSV (180s cutoff): `*_desc-cosine180s_confounds.tsv` (generated by `make_cosine_basis.py`)
- Potentially other confound TSVs later (AROMA noise components, etc.)

Choosing **which confounds to include** in the nuisance regression is a non-trivial decision that benefits from visual inspection (e.g. are two confounds highly correlated? does adding AROMA components overlap with aCompCor?). A command-line script can't do this interactively.

The **fMRI Denoiser** is an R Shiny app that:
1. Lets the user browse the server filesystem and pick a derivatives folder
2. Discovers all BOLD files and confound TSVs within it
3. Lets the user choose which tasks, confound files, and confound columns to use
4. Shows the correlation matrix of selected confounds as a QA heatmap
5. Saves the configuration to a JSON file
6. **Generates two outputs**:
   - Per-subject/run confound `.txt` files (pre-built, ready for `fsl_regfilt`)
   - A clean, readable bash script `run_regfilt.sh` that just calls `fsl_regfilt` in parallel

The app lives on the Shiny server so participants can access it from a browser without any local installation.

## Key design decisions (updated 2026-04-17)

| Decision | Details |
|---|---|
| **File placement** | Confound `.txt` files alongside BOLD (in fmriprep derivatives); config JSON + `run_regfilt.sh` in user-selected output folder |
| **Task selection only** | User selects tasks; all runs for a task are automatically included — no run picker |
| **Subject-level parallelism** | `xargs -P` processes N subjects simultaneously; each subject's BOLD files run sequentially inside the function |
| **Runtime `find`** | Script uses `find` at runtime to locate BOLDs per subject; task filter lines embedded by the app |
| **Config saves output_dir** | Loading a config also restores the output folder path |
| **shinyFilesButton for load** | Server-side file picker for loading existing `.json` config |
| **All tasks/TSVs pre-selected** | Tables open with all rows selected — user deselects what they don't want |

## Column 3 — QA heatmap

A correlation matrix heatmap of all selected confound columns, computed from the **first available subject** (or averaged across subjects — simpler to start with one).

- X/Y axes: selected column names
- Color: correlation coefficient (−1 to +1)
- Purpose: spot multicollinearity (e.g. if two aCompCor components are highly correlated, one may be redundant; if motion and cosine regressors are correlated, the HPF cutoff may be too short)

Updates reactively when confound column selection changes.

```r
# Example with pheatmap
corr_matrix <- cor(confounds_df, use = "pairwise.complete.obs")
pheatmap(corr_matrix, ...)
```

## Output 1: Per-subject confound `.txt` files

For **each** BOLD file matching the selected tasks, the app:

1. Reads all selected confound TSVs for that subject/session/task/run
2. Selects only the chosen columns
3. `bind_cols()` across TSVs
4. `mutate(across(everything(), ~replace_na(.x, 0)))` — fills NaN (first row of derivative columns) with 0
5. Writes a space-separated, no-header `.txt` file alongside the BOLD:

```
sub-XX_ses-01_task-rest_space-MNI152NLin2009cAsym_res-2_desc-selected_confounds.txt
```

The filename mirrors the BOLD file exactly (replacing `_desc-preproc_bold.nii.gz` with `_desc-selected_confounds.txt`) so the bash script can find it with a simple string substitution.

## Output 2: `run_regfilt.sh`

A clean, self-documenting bash script saved to the derivatives folder. The user reads it, checks it, then runs it.

```bash
#!/bin/bash
# ============================================================
# Confound regression with fsl_regfilt
# Generated by fMRI Denoiser app — 2026-04-16 17:00
# ============================================================
#
# Derivatives   : /data03/MRI_hackaton_data/Data_collection/fmriprep
# Task(s)       : rest
#
# Confound files used:
#   *_desc-confounds_timeseries.tsv    (fMRIprep)
#   *_desc-cosine180s_confounds.tsv    (cosine HPF 180s)
#
# Selected regressors (28 total):
#   trans_x  trans_y  trans_z  rot_x  rot_y  rot_z
#   trans_x_derivative1  trans_y_derivative1  trans_z_derivative1
#   rot_x_derivative1  rot_y_derivative1  rot_z_derivative1
#   trans_x_power2  trans_y_power2  trans_z_power2
#   rot_x_power2  rot_y_power2  rot_z_power2
#   trans_x_derivative1_power2  ...  rot_z_derivative1_power2
#   a_comp_cor_00  a_comp_cor_01  a_comp_cor_02  a_comp_cor_03  a_comp_cor_04
#   a_comp_cor_05  a_comp_cor_06  a_comp_cor_07  a_comp_cor_08  a_comp_cor_09
#   cosine_hpf_01  cosine_hpf_02  cosine_hpf_03
#   cosine_hpf_04  cosine_hpf_05  cosine_hpf_06
#
# Confound .txt files were pre-built by the app.
# If you change the selection, re-run the app to regenerate them.
#
# To run:
#   cd /data03/MRI_hackaton_data/Data_collection/fmriprep
#   nohup bash run_regfilt.sh > run_regfilt.log 2>&1 &
# ============================================================

deriv_dir="/data03/MRI_hackaton_data/Data_collection/fmriprep"
n_parallel=5

find "${deriv_dir}" \
    -name "*_task-rest_*_desc-preproc_bold.nii.gz" | \
xargs -n1 -P${n_parallel} -I{} bash -c '

    bold="{}"
    confounds="${bold/_desc-preproc_bold.nii.gz/_desc-selected_confounds.txt}"
    output="${bold/_desc-preproc_bold.nii.gz/_desc-denoised_bold.nii.gz}"

    # number of regressors → regress them all out
    n_cols=$(awk "NR==1{print NF}" "${confounds}")
    indices=$(seq 1 ${n_cols} | paste -sd,)

    echo "Processing: $(basename ${bold})"
    fsl_regfilt \
        -i "${bold}" \
        -d "${confounds}" \
        -f "${indices}" \
        -o "${output}"
'
```

## Running and stopping `run_regfilt.sh`

### Launch

```bash
nohup bash run_regfilt.sh > run_regfilt.log 2>&1 &

# Monitor progress
tail -f run_regfilt.log
```

### Stop everything (if you made a mistake)

Because `xargs` spawns child processes that won't automatically die when the parent is killed, use two commands:

```bash
# Kill the main script + xargs workers
pkill -f run_regfilt.sh

# Kill any fsl_regfilt processes already running
pkill -f fsl_regfilt

# Verify nothing is left
pgrep -f fsl_regfilt   # should return nothing
```

**Or kill the whole process group in one shot:**
```bash
kill -- -$(pgrep -o -f run_regfilt.sh)
```
`-$(...)` sends SIGTERM to the entire process group; `-o` picks the oldest process (the group leader).

### What happens if a subject has a different set of tasks?

The script handles heterogeneous datasets gracefully. For each subject, `find` returns only whatever BOLD files actually exist. If `sub-007` has no `task-trust` BOLD, `find` simply returns nothing for that task and moves on. The confound `.txt` files written by the app and the BOLDs processed by the script are always in sync because they were derived from the same file discovery.

## Notes on `fsl_regfilt`

`fsl_regfilt` regresses out specified components from a 4D BOLD image:
```bash
fsl_regfilt -i <bold.nii.gz> -d <confounds.txt> -f "1,2,...,N" -o <output.nii.gz>
```
- `-d`: space-separated confound matrix (rows = timepoints, cols = regressors), **no header**
- `-f`: comma-separated list of column indices to regress out (1-based)
- The output retains the mean signal (it only removes the specified components, not the intercept)
- All `N` columns are always regressed out (we don't keep any "good" columns)



## Partial vs full (aggressive) regression

Let $Y$ be the observed data and $X = [x_1, \dots, x_p]$ the design matrix.

The standard linear model is:

$$ Y = X\beta + \varepsilon $$

with least-squares solution:

$$ \hat{\beta} = (X^\top X)^{-1} X^\top Y $$

and fitted signal:

$$ \hat{Y} = X\hat{\beta} $$

so the aggressive (“full regression”) cleaned data is:

$$ Y_{\text{clean}} = Y - \hat{Y} = (I - P_X)Y $$

where the projection matrix onto the column space of $X$ is:

$$ P_X = X(X^\top X)^{-1}X^\top $$

In this formulation, all variance in $Y$ explained by any linear combination of regressors
is removed in a single orthogonal projection.

------------------------------------------------------------

In the “partial” formulation, the same fitted model is implicitly decomposed into
contributions associated with each regressor after removing overlap with the others.

For each regressor $x_i$, define its residualized version with respect to the remaining
regressors $X_{-i}$:

$$ x_i^\perp = M_{-i} x_i $$

where the residual-forming matrix is:

$$ M_{-i} = I - X_{-i}(X_{-i}^\top X_{-i})^{-1}X_{-i}^\top $$

This removes from $x_i$ any component explainable by the other regressors.

------------------------------------------------------------

Each regressor then contributes via its own projection operator:

$$ P_i = x_i^\perp (x_i^{\perp\top} x_i^\perp)^{-1} x_i^{\perp\top} $$

yielding partial fitted contributions:

$$ \hat{Y}_i = P_i Y $$

The total reconstructed signal is:

$$ \hat{Y}_{\text{partial}} = \sum_{i=1}^p \hat{Y}_i $$

and the cleaned data becomes:

$$ Y_{\text{clean}} = Y - \hat{Y}_{\text{partial}} $$

------------------------------------------------------------

This construction ensures that each regressor contributes only through its uniquely
identifiable component after orthogonalization with respect to the remaining design space,
avoiding ambiguous allocation of shared variance inherent to the full-model coefficients
$\hat{\beta}$.
